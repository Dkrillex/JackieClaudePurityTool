# 资源纯度测试 · Streamlit 可视化界面

为 `../purity/`（聚合平台 / API 中转资源纯度测试核心库）提供网页版界面：
在浏览器里配置渠道、勾选探针、一键运行，实时查看纯度对照评分并导出报告。

## 运行

```bash
cd relay_purity_ui
pip install -r requirements.txt
streamlit run app.py
```

浏览器会自动打开 `http://localhost:8501`。

## 功能

- **可配置渠道表格**：直接增删/编辑渠道（名称、Base URL、API Key、模型、是否基准），也可上传 `providers.json` 一键导入。
- **可选探针**：左侧勾选要运行的维度（协议纯度 / 模型新鲜度 / 推理 / 指令忠实 / 流式 / 凭据安全），并可调单请求超时。
- **对照评分**：彩色总览表（总分按纯净 / 基本纯净 / 掺水 / 严重掺假着色）+ 逐渠道明细。
- **导出报告**：一键下载 Markdown 报告与 CSV 评分表。

## 评分维度（满分 100）

| 维度 | 权重 | 含义 |
|------|------|------|
| 协议纯度 | 25 | 原生 Anthropic `/v1/messages` + OpenAI 是否真可用 |
| 模型新鲜度 | 25 | 客观知识探针，排除降智到老模型 |
| 推理完整性 | 20 | 经典陷阱题，弱/量化模型常翻车 |
| 指令忠实度 | 15 | 是否被隐藏系统提示覆盖 |
| 流式完整性 | 10 | SSE 结构是否正常、模型名是否一致 |
| 凭据/错误安全 | 5 | 错误响应是否泄露密钥/上游 |

> 方法论：只用可客观验证的证据打分；模型「自报身份」不计分（幻觉，仅作记录）。

## 注意

- 测试需要填入真实 API Key，请勿把含密钥的 `providers.json` 提交进仓库。
- 依赖系统 `curl` 作为传输层（与命令行工具一致）。
